## 一、战略Agent（Strategic Agent）

**定位**：军区司令——只负责方向性决策，不碰具体操作。

### 唤醒条件

| 触发源           | 条件                      | 说明                             |
| ------------- | ----------------------- | ------------------------------ |
| **定时唤醒**      | 每30秒（可配置）               | 常规战略评估周期                       |
| **玩家战略指令**    | 玩家指令改变了战略方向             | 由调度Agent唤醒，并注入Player Directive |
| **调度Agent反馈** | 重大任务完成/失败/战略目标达成/异常事件上报 | 需要重新评估战略方向                     |

### 输入

| 输入项           | 来源            | 说明                 |
| ------------- | ------------- | ------------------ |
| **历史操作链**     | 自身历史          | 压缩后的历史摘要，形成战略连贯性   |
| **战场态势**      | 调度Agent（共享状态） | 全局态势（兵力/经济/科技/建筑）  |
| **上一轮输出**     | 自身            | 上一轮的战略清单           |
| **调度Agent反馈** | 调度Agent       | 任务执行状态、资源消耗、异常报告   |
| **玩家指令（战略级）** | 调度Agent注入     | 如“转为防守”、“目标改为科技建筑” |
| **玩家指令（微操级）** | 不接收           | 微操不进入战略Agent       |

### 输出

```json
{
  "strategy_id": "S_001",
  "timestamp": 12345,
  "direction": "aggressive_expansion",
  "priorities": [
    {"item": "economic", "weight": 0.3},
    {"item": "army_building", "weight": 0.4},
    {"item": "aggressive_pressure", "weight": 0.3}
  ],
  "global_objectives": [
    {"id": "obj_001", "type": "destroy", "target": "enemy_construction_yard", "priority": 1}
  ],
  "threat_assessment": {
    "main_threat": "enemy_air_force",
    "vulnerable_points": ["enemy_mining", "enemy_tech_building"]
  },
  "resource_allocation": {
    "production_focus": "HTNK",
    "reserve_ratio": 0.2
  }
}
```

### 交互关系

| 方向      | 对象      | 内容                 |
| ------- | ------- | ------------------ |
| **输出→** | 调度Agent | 战略清单（只读，不包含任何指令）   |
| **←输入** | 调度Agent | 执行反馈、任务状态、玩家战略指令摘要 |
| **←输入** | 共享状态    | 战场态势（只读）           |

## 二、调度Agent（Dispatch Agent）

**定位**：前线指挥部——把战略拆解成具体作战任务，安排资源，响应紧急事件。

### 唤醒条件

| 触发源          | 条件           | 说明               |
| ------------ | ------------ | ---------------- |
| **定时唤醒**     | 每5秒（可配置）     | 常规调度周期           |
| **新战略清单**    | 战略Agent更新    | 需要重新拆解任务         |
| **玩家指令（任何）** | 任何时候         | 全局指令立即响应，不能等定时周期 |
| **Team状态变化** | Team完成/失败/异常 | 需要调整任务或回收资源      |
| **紧急事件**     | 基地被攻击、主力被全歼等 | 需要立即全局响应         |

### 输入

| 输入项           | 来源      | 说明                |
| ------------- | ------- | ----------------- |
| **战略清单**      | 战略Agent | 当前战略方向和目标         |
| **战场态势**      | 共享状态    | 全局态势（兵力/经济/科技/建筑） |
| **上一轮输出**     | 自身      | 上一轮的任务列表和Team状态   |
| **介入Agent状态** | 介入Agent | 各Team的执行进度和状态     |
| **玩家指令（所有）**  | 玩家      | 包括微操和战略指令，都要立即响应  |
| **产线状态**      | 共享状态    | 当前生产队列、空闲情况       |

### 输出

```json
{
  "dispatch_id": "D_001",
  "timestamp": 12345,
  "teams": [
    {
      "team_id": "T_001",
      "status": "active",
      "units": ["0x1234", "0x5678"],
      "script": "AttackBase",
      "target": {"type": "Building", "address": "0xabcd"},
      "priority": 5,
      "removable": true,
      "intervention_agent": "IA_001"
    }
  ],
  "production_orders": [
    {"type": "HTNK", "count": 6, "priority": 5}
  ],
  "emergency_override": null,  // 如果存在，则覆盖所有Team
  "strategic_feedback": {
    "completed": ["T_001"],
    "failed": [],
    "pending": ["T_002"]
  }
}
```

### 交互关系

| 方向      | 对象      | 内容             |
| ------- | ------- | -------------- |
| **←输入** | 战略Agent | 战略清单           |
| **→输出** | 战略Agent | 执行反馈           |
| **→输出** | 介入Agent | Team创建/销毁/任务指派 |
| **←输入** | 介入Agent | 执行进度、异常报告      |
| **→输出** | 玩家      | 指令执行确认（如需要）    |
| **←输入** | 玩家      | 所有指令（直接响应）     |
| **↔双向** | 共享状态    | 读写全局状态         |

## 三、介入Agent（Intervention Agent）

**定位**：班长——带领指定单位执行具体任务，平时休眠，只在被唤醒时工作。

### 唤醒条件

| 触发源           | 条件        | 说明             |
| ------------- | --------- | -------------- |
| **调度Agent唤醒** | 创建新Team时  | 初始化任务          |
| **调度Agent唤醒** | 任务变更时     | 修改目标/路径/策略     |
| **调度Agent唤醒** | 任务异常时     | 卡住/重大损失/需要人工干预 |
| **玩家指令**      | 直接指向该Team | 通过调度Agent转发    |

### 输入

| 输入项            | 来源        | 说明               |
| -------------- | --------- | ---------------- |
| **操控范围**       | 调度Agent   | 分配的单位列表（地址列表）    |
| **当前任务**       | 调度Agent   | 任务目标、Script、优先级  |
| **上一轮输出**      | 自身        | 上一轮的指令和执行反馈      |
| **当前情况**       | 共享状态      | 本队单位状态、附近敌情      |
| **任务流（历时列表）**  | 调度Agent   | 从最初任务到当前修改的完整指令链 |
| **玩家指令（针对本队）** | 调度Agent转发 | 如“第一队绕后”         |

### 输出

```python
# 微操指令序列
[
    {"type": "IssueOrder", "unit": "0x1234", "mission": "Move", "target": {"X": 10, "Y": 20}},
    {"type": "IssueOrder", "unit": "0x5678", "mission": "Attack", "target": "0xabcd"},
    {"type": "QueueMission", "unit": "0x1234", "mission": "Guard"},
]

# 执行反馈
{
    "team_id": "T_001",
    "status": "executing",
    "progress": "已摧毁2座防御塔",
    "remaining_units": 5,
    "lost_units": 1,
    "expected_completion": 12360,
    "issues": ["遭遇敌方IFV拦截", "路径被阻塞"]
}
```

### 交互关系

| 方向      | 对象      | 内容                            |
| ------- | ------- | ----------------------------- |
| **←输入** | 调度Agent | Team分配、任务指派、唤醒指令              |
| **→输出** | 调度Agent | 执行反馈、异常报告                     |
| **←输入** | 共享状态    | 本队单位状态、附近敌情（只读）               |
| **→输出** | 执行层（引擎） | 微操指令（IssueOrder/QueueMission） |

## 五、交互关系速查表

| 关系         | 方向 | 内容             | 频率       |
| ---------- | -- | -------------- | -------- |
| **战略→调度**  | 输出 | 战略清单           | 每30秒     |
| **调度→战略**  | 输出 | 执行反馈           | 事件驱动     |
| **调度→介入**  | 输出 | Team创建/销毁/任务指派 | 事件驱动     |
| **介入→调度**  | 输出 | 执行进度/异常报告      | 事件驱动     |
| **玩家→调度**  | 输入 | 所有指令           | 事件驱动     |
| **调度→玩家**  | 输出 | 执行确认           | 事件驱动     |
| **战略↔共享**  | 读  | 态势信息           | 每30秒     |
| **调度↔共享**  | 读写 | 全局状态           | 每5秒/事件驱动 |
| **介入↔共享**  | 读  | 本队相关信息         | 事件驱动     |
| **介入→执行层** | 输出 | 微操指令           | 事件驱动     |

## 六、总结

| Agent       | 唤醒频率              | 核心输入                | 核心输出           | 权限          |
| ----------- | ----------------- | ------------------- | -------------- | ----------- |
| **战略Agent** | 30秒定时 + 调度反馈/战略指令 | 历史+态势+调度反馈+玩家战略指令   | 战略清单           | 只读，不操作      |
| **调度Agent** | 5秒定时 + 任何事件       | 战略清单+态势+Team状态+玩家指令 | Team任务+产线安排+反馈 | 可查一切，可下全局指令 |
| **介入Agent** | 被调度Agent唤醒        | 操控范围+任务+任务流         | 微操指令+执行反馈      | 仅操作分配的单位    |

**关键原则**：

- 战略Agent只输出“意图”，不输出“指令”
- 调度Agent在“全局指令”时可跨Team操作，其余时间只操作Team
- 介入Agent只操作分配的单位，且被调度Agent约束
- 玩家指令同时进入战略Agent（异步）和调度Agent（同步）
